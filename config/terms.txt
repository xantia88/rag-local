Параметр `title` используется для указания наименования автоматизированной системы или функциональной подсистемы. Наименование должно быть кратким и отражать суть системы или подсистемы без излишнего описания.
Параметр `description` используется для развернутого описания автоматизированной системы/функциональной подсистемы. Он может содержать наименование производителя, обобщенное описание выполняемых функций, специфику применения и любую другую информацию, которая может характеризовать систему.
Параметр `class` используется для указания класса системы согласно принятой классификации. В качестве значений данной аналитики должны использоваться значения соответствующего справочника.
Параметр `group` используется для указания идентификатора группы, в которую включается автоматизированная система или функциональная подсистема.
Параметр `parent` (родительский) — это аналитика, используемая для указания идентификатора автоматизированной системы, которая является родительской для данной функциональной подсистемы. Родительская система представляет собой систему, которая предоставляет функциональность, необходимую для работы данной функциональной подсистемы. Это позволяет создавать и поддерживать структуру подчиненности и зависимостей между различными компонентами ИТ-инфраструктуры.
Параметр `location` используется для указания места нахождения автоматизированной системы по отношению к периметру ИТ-ландшафта Компании. Системы, находящиеся вне периметра ИТ-ландшафта Компании, считаются внешними, соответственно, системы, находящиеся внутри периметра Компании — внутренними.
Параметр `Степень владения` используется для указания правового аспекта использования системы. Значения этого параметра могут быть следующими: "Владеем исходным кодом", "Владеем экземпляром", "SaaS", "Система партнера", "Владеем экземпляром". Этот параметр обязателен для систем, находящихся на определенных этапах жизненного цикла, и помогает определить, кто именно управляет и поддерживает систему.
Параметр `target-status` используется для указания целевого утвержденного статуса автоматизированной системы или функциональной подсистемы.
Параметр `Live-stage` (этап жизненного цикла) — это аналитика, используемая для указания текущего этапа жизненного цикла автоматизированной системы или функциональной подсистемы. Эта аналитика помогает отслеживать состояние системы на протяжении всего ее жизненного цикла, начиная с этапа эскиза и заканчивая архивной стадией.
Параметр `live-stage-target` определяет целевой этап жизненного цикла автоматизированной системы или функциональной подсистемы на горизонт планирования. Этот параметр используется для описания того, каким образом система будет развиваться в будущем, учитывая запланированные изменения и этапы ее развития.
Параметр `change-type` используется для указания действия, производимого над автоматизированной системой или функциональной подсистемой для достижения целевого этапа жизненного цикла. Возможные значения включают: "Планируется", "Разрабатывается", "Модифицируется", "Используется", "Выводится из эксплуатации".
Параметр `criticality` определяет уровень критичности автоматизированной системы или функциональной подсистемы, который отражает степень влияния сбоя или отказа этой системы на бизнес-процессы компании. Критичность может варьироваться от Mission Critical (критически важная) до Office Productivity (продуктивность офисной работы), в зависимости от роли системы в организации.
Параметр `performance` используется для указания подтвержденных параметров производительности системы. Он может содержать информацию о количестве запросов, времени обработки одного запроса, времени чтения, записи, вставки и удаления объектов данных в системе или иные показатели, характеризующие производительность. Для внешних систем, обычно, производительность фиксируется в Договоре об уровне сервиса (SLA).
Параметр `RTO` (Recovery Time Objective) — это максимальный период времени, в течение которого система гарантированно должна быть восстановлена после сбоя или выхода из строя. Этот показатель измеряется в часах и используется для определения допустимого времени простоя системы при аварии.
Параметр `RPO` (Recovery Point Objective) — это максимальный период времени, в течение которого могут быть потеряны данные в результате сбоя или выхода из строя автоматизированной системы или функциональной подсистемы. Этот показатель измеряется в часах и отражает допустимый объем возможных потерь информации перед восстановлением системы.
Параметр `SLA` (Service Level Agreement) — это соглашение об уровне услуги, в котором определяются параметры качества обслуживания, которые провайдер услуги гарантирует клиенту в течение определенного периода времени. Эти параметры могут включать уровень доступности услуги, время отклика, скорость обработки запросов и другие показатели, важные для функционирования бизнес-процессов.
Параметр `monitoring` используется для указания систем мониторинга, применяемых в автоматизированной системе или функциональной подсистеме.
